This topic provides an overview on how to migrate from using HapYak to using Brightcove Interactivity. If you are just getting started with Brightcove Interactivity, check out Getting Started with the Interactivity Module.

Migrating content published through the Brightcove/HapYak plugin

If your HapYak projects were generated using the HapYak/Brightcove plugin and your content is currently deployed as Brightcove embed code/URL, the process to transition customer-facing interactive elements from the HapYak instance to the Brightcove Interactivity version is simple. Once the HapYak content migration occurs, follow these steps to update your customer-facing content to the new Brightcove versions.

  1. Enable Brightcove Interactivity on any players you are currently using to serve HapYak-enabled interactivity.
    • Open the Players module
    • Click on a Name
    • In Additional Features > Interactivity, check the Enable Interactivity option
    Note: For more information, see Configuring a Brightcove Player with the Interactivity Plugin.

    The checkbox to Enable Interactivity will replace the need for entering any API key data.

  2. If any custom tracking scripts were being used, enable the Options (JSON) checkbox and enter the script.
  3. (Optional) If you would like to have projects auto-created when they are played with an Interactivity-enabled Brightcove Player (similar to functionality from the HapYak/Brightcove plugin), you will need to enable project auto-creation on your player. To do this, add the following to the Options(JSON).
  4. Delete the existing hapyak plugin on the player (click Plugins in the navigation).
  5. Save the changes and publish the player.

After caching clears, the Brightcove Interactivity versions of the interactive content should automatically replace the HapYak versions. Customer-facing experiences should continue functioning seamlessly on the new platform.

Migrating non-Brightcove plugin published projects

Note: To facilitate a smooth update process, we recommend that you upload versions of any non-Brightcove videos used as source files for HapYak content into Video Cloud before starting this process.

If your interactive content was not published using a Brightcove embed code via the HapYak plugin (typical examples include direct publishing from HapYak embed code and non-Brightcove based projects) you will still have a path to easily update your content. During the HapYak content migration, Brightcove Interactivity versions of all your interactive content will be generated. In cases where a non-Brightcove video was used as the video source file, the migration will create a "headless" project retaining all of your interactive content. You will be required you to update the project settings and select a valid Brightcove video source file (that exists in the Media module).

  1. Open the Interactivity module.
  2. Click on a project Title. Projects with a missing video will display a warning icon as the video thumbnail.
  3. Click Settings.
  4. Click Replace Video.
  5. Select a video and click Replace.
    Note: For more information on project settings, see Configuring Project Settings.
  6. Verify the timing and functionality of any interactive elements in the annotation editor or by previewing the project.

Migrating HapYak experiences

Brightcove Interactivity supports interactive experiences natively. As part of the HapYak content migration, versions of your experiences will be created and be available within the Interactivity module. Each experience will have its own unique embed code. You will need to replace any existing HapYak experience embed code with the Brightcove Interactivity embed code on your sites.

  1. Open the Interactivity module.
  2. In the left navigation, click Experiences.
  3. Find the experience using search or by scrolling through the list of experiences.
  4. Verify that the experience functionality works as expected and that the appropriate settings are configured.
  5. Click Embed.
  6. Copy the embed code for the experience.
  7. Replace the existing HapYak embed code with the new Brightcove Interactivity embed code on your site.
Note: If you have any HapYak experiences with videos originating across multiple Brightcove accounts, any videos whose video source file is not present in the account that was migrated to will be in a "headless" project state. Projects without videos will display a warning icon for the video thumbnail. These projects will need to have the source video replaced with a video from that account.
Note: Due to the greater suite of capabilities available within Brightcove Interactivity, any landing page experiences should be embedded from the individual project level. Open the project, click Embed and then click Get URL to generate a direct page for the content.
Note: Any SCORM packages will need to be re-downloaded and redeployed to your LMS. Open the experience, click Embed, click the SCORM tab, select the SCORM version and then click Download.
